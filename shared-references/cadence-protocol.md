# Cadence Protocol（节奏协议）

被 `vibe-status`、`vibe-recommend`、SessionStart hook 引用。

固化"什么时候该做什么"——避免用户驱动每一步。

---

## WIP 警戒规则

**WIP = `state.wip_tasks` 数组长度** = 已评估但未完成的任务数。

| WIP 数量 | 状态 | 行动 |
|---|---|---|
| 0 | 🔴 空 | 需要评估新任务 |
| 1-2 | 🟢 正常 | 节奏稳定 |
| 3-4 | 🟠 偏高 | 建议先完成已有任务再开新任务 |
| 5+ | 🔵 积压 | **暂停开新任务**，全力完成已有 WIP + 复盘 |

---

## 复盘提醒

任何任务完成但未复盘 → `vibe-status` / SessionStart 顶部高亮提醒。

---

## 任务推荐策略（vibe-recommend）

每次推荐 ≥2 个任务时：

### 第 1 个（稳分）
- 综合分高 + CS 高（需求清晰）
- 与最近完成的任务模块不重复

### 第 2 个（实验性）
- 能验证某个待验证假设
- 综合分不一定 top，但有"信息价值"

### WIP 状态对推荐的覆盖

| WIP 状态 | 推荐策略 |
|---|---|
| 🔴 空 | 标准 1+1 |
| 🟢 正常 | 标准 1+1 |
| 🟠 偏高 | 只推稳分 top 1 |
| 🔵 积压 | **暂停推荐**，提醒先完成 WIP + 复盘 |

---

## 节奏元规则（优先级从高到低）

1. **WIP 优先于评分**：积压时不要因为"等更好的任务"而开新任务
2. **复盘优先于新评估**：完成任务后**先复盘再评估新任务**
3. **实验性最多 1/天**：每次评估 ≥1 个稳分任务
