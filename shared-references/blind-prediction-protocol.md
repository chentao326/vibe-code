# Blind Prediction Protocol（盲预估协议）

被这些子 skill 引用：`vibe-assess`、`vibe-retro`、主 SKILL.md。

这是项目原则 #1 的完整规范。任何子 skill 在写预估前都必须执行本协议。

---

## 核心定义

**盲预估**：在开始编码**之前**完成的预估。预估者（人或 AI）不得见过任何执行结果数据。

预估一旦写入 `predictions/*.md` 的 `## 预估 v1` 段，该段即为 **immutable**——只能在文件末尾追加 `## 复盘` 段，不能修改预估段任何字符。

---

## "见过数据"的边界

下列任一条件成立 → 已不再 blind，**禁止写预估**：

| 信息 | 是否破坏 blind | 例外 |
|---|---|---|
| 已开始编码（任何程度） | ✗ 破坏 | 无 |
| 已看过 git diff / git log 结果 | ✗ 破坏 | 无 |
| 已跑过 lint / test 并看到结果 | ✗ 破坏 | 无 |
| 已知实际耗时 | ✗ 破坏 | 无 |
| 已跟 AI 进行过相关对话 | ✗ 破坏 | 无 |
| 类似任务的历史数据 | ○ 不破坏 | 这是锚点对比的依据 |
| 用户说"这个应该简单" | △ 谨慎 | 主观偏见，需标注 |

**判断捷径**：只要这条信息**只能在编码执行后才能获得**，就算"数据"。

---

## 预估者必须主动声明的情况

`vibe-assess` 在启动前，必须自检并向用户**主动声明**：

1. **任务已开始编码** → 必须拒绝写"预估"，改记为 `**Reconstructed retrospective**`
2. **任务已完成但用户尚未透露任何数据** → 罕见；允许但标 `published_before_prediction: true`
3. **用户在对话里已提到任何执行结果** → 同 #1 处理

`BLIND_CHECK=strict`（默认）：任何破坏条件命中 → **拒绝执行**。
`BLIND_CHECK=lenient`：仅警告 + 强制标注 → 仅用于离线演练。

---

## Immutable 的工程边界

`## 预估 v1` 段的不可修改是**用户体验承诺**，由 hook 层强制：

- `hooks/prediction-immutability.sh` 在 PreToolUse(Edit|Write) 上检查 `predictions/` 下文件
- 命中 `## 预估 v1` 与 `## 复盘` 之间任何 diff → exit 1 阻塞
- `## 复盘` 段的追加 → 放行

**禁止的"绕开"模式**（子 skill 必须拒绝）：
- "把预估改得更准一点" → 拒绝。创建 `_redo.md` 新文件
- "我的耗时写错了" → 拒绝。在复盘段追加修正说明
- "我前面没考虑这个维度" → 拒绝。同上路径

唯一允许编辑预估段的场景：**纯 markdown 排版错误**，且用户显式 bypass。

## 文件名约定（三处一致）

一个任务三处文件，**用同一组 `<date>_<id>_<short>` 命名**：

```
tasks/<date>_<id>_<short>.md          ← 任务描述
predictions/<date>_<id>_<short>.md    ← immutable 预估
retros/<date>_<id>_<short>/           ← 复盘数据
  ├── report.md                       ← 自动采集的数据快照
  └── notes.md                        ← 主观复盘笔记
```

- `<date>`：任务描述首次落盘日期
- `<id>`：12 位 sha256 前缀，对**任务描述首次落盘的内容**做 hash
- `<short>`：3-8 字英文短名

## 子 skill 检查清单

`vibe-assess` 启动时：
1. 读 `BLIND_CHECK` 常量
2. 询问用户该任务当前执行状态（未开始 / 进行中 / 已完成）
3. 询问对话历史里是否提到过任何执行结果
4. 若命中破坏条件 → 按 `BLIND_CHECK` 模式处理
5. 通过后才允许写 `predictions/*.md`

`vibe-retro` 启动时：
1. 读目标 prediction 文件
2. **先在内存里 cache 住 `## 预估 v1` 段**
3. 采集数据 → 追加 `## 复盘` 段
4. 写完后**再次校验**：hash 应等于步骤 2 的 cache

---

## 反模式（必须拒绝的请求）

- 「帮我预估一下，但我先告诉你实际耗时你反推就行」 → 拒绝
- 「这条已经做完了，但假装没看到给我预估看准不准」 → 拒绝。走 `_redo.md` reconstructed 路径
- 「上次预估算错了帮改一下」 → 拒绝。在复盘段说明
