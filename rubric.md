## v1 综合分公式

```
composite = (CS×1.0 + CX×1.5 + TE×1.5 + AM×1.0 + AQ×1.0) / 6.0 × 2.0
```

每个维度 0-5 整数分。综合分范围 0-10。

---

## 5 个维度

### CS — Clarity of Spec（需求清晰度）×1.0
*任务描述有多清晰、具体、可执行？*

- **0** — "帮我改个bug"（完全无法执行）
- **3** — 描述了问题现象和期望结果，但缺少具体文件/行号
- **5** — 精确到文件路径+函数名+期望行为+验收标准

### CX — Cross-cutting Impact（改动影响面）×1.5
*这个任务涉及多少模块/文件/系统？*

- **0** — 单文件，纯函数，无外部依赖
- **3** — 同模块内 3-5 个文件
- **5** — 跨 5+ 模块，涉及数据库 schema + API + 前端

### TE — Testability（可验证性）×1.5
*多容易判断"做对了"？*

- **0** — 结果纯主观，无法自动验证
- **3** — 有明确的验收标准但无自动测试
- **5** — 有完整的自动化测试覆盖 + 一行命令可验证

### AM — Ambiguity（隐性知识占比）×1.0
*多少信息在脑子里没写进任务描述？*

- **0** — 所有信息已写在 task 描述中
- **3** — 需要一些业务背景知识，但可推断
- **5** — 大量业务逻辑和设计决策只在用户脑子里

### AQ — Agent Quality Match（AI 匹配度）×1.0
*这个任务是否适合 AI 执行？*

- **0** — 需要 AI 工具链不支持的能力（如 GUI 操作）
- **3** — AI 能做但需要多轮澄清
- **5** — AI 原生强项：代码生成、重构、bug 修复

### 2026-05-26 Spearman ρ = −0.893 负相关诊断 — 需在 v2 前验证方向

- **现象**: v0→v1 bump 时计算的 Spearman ρ = −0.893，绝对值超过 0.80 阈值，但方向为负
- **含义**: 当前公式下，高分任务实际耗时/轮次反而更少——综合分与任务难度可能反向
- **两种可能原因**:
  1. **小样本偏差**: 仅 6 个样本，单一异常值可翻转方向。ρ = −0.893 的绝对值在如此小的 N 下不稳定
  2. **公式方向倒置**: 维度设计可能更接近"AI 适配度"而非"任务难度"——高分任务恰是 AI 擅长、完成更快的
- **v2 升级前需验证**:
  - 积累到 N ≥ 12 个盲预估样本后重新计算 Spearman ρ
  - 若 ρ 仍为负且绝对值 > 0.7，则 v2 需考虑维度重新定向或方向取反
  - 若 ρ 转正，则确认为小样本偏差，当前公式方向可用

- 样本量 6，仍偏小——随校准池扩大需持续监控 ρ 稳定性
